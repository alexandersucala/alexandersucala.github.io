Finance Assistant V2 Live

Fail-closed AI stock analysis with compliance gates. Implements Fail-Closed Execution, State-Locked AI Orchestration, Artifact-Based Authority, and Intent-Contract Enforcement patents in a production market intelligence system.

Why it matters: Most AI systems fail open — producing output regardless of data quality. This system fails closed, gating all responses behind a three-stage LLM pipeline with dual compliance gates and three-dimensional confidence scoring.

Pipeline Architecture

User Query ↓ [Gate A] Intent Classifier — blocks advisory requests (regex fast-path + LLM fallback) ↓ [Data Fetch] Alpha Vantage (market) + Brave Search (news) ↓ [Stage 1] Draft Builder — structured analysis with citation refs (Sonnet) ↓ [Stage 2] Delivery Filter — transforms to natural language (Haiku) ↓ [Stage 3] Policy Scanner — deterministic + LLM compliance check (Haiku) ↓ [Gate B] Output Validation — PASS / REWRITE / FAIL_CLOSED ↓ Final Report + Trace Log

Analysis Quality Scores

Replaces single "confidence" with three independent dimensions displayed on every report:

Data Quality Source freshness, data completeness, timestamp coherence
Compliance Policy scan result — PASS / REWRITE / FAIL_CLOSED
Sources Citation count, support strength, unsupported claim penalties

CodeReview AI In Development

Automated, document-grounded code review that integrates directly with GitHub. The system auto-discovers a repository's documentation, classifies it into review categories, then uses that knowledge base to flag security violations, architectural drift, onboarding process failures, and style issues — all with exact documentation citations.

Why it matters: Generic AI code review gives opinions. This system gives document-backed findings — every flag references the company's own docs, not general best practices. Findings are tagged DOCUMENT_BACKED or LLM_OPINION so reviewers know what's policy and what's suggestion.

Review Pipeline

GitHub PR Webhook ↓ [/setup] Auto-Discovery — scans repo tree, fetches docs, classifies by gate ↓ [Decomposer] Splits large docs into gate-specific sections (LLM + keyword fallback) ↓ [Pass 1] Four parallel gate reviews: ├── Security — auth, injection, secrets, data exposure ├── Architecture — patterns, boundaries, coupling, API contracts ├── Onboarding — PR format, checklists, test requirements, commit style └── Style — naming, formatting, linting, import order ↓ [Pass 2] Hallucination Verification — re-checks each finding against source docs ↓ [Traffic Light] RED (blocking) / YELLOW (needs fixes) / GREEN (ready) ↓ PR Comment with cited findings

Classification Pipeline

  • Step 1 Filename heuristics — fast pattern matching (~80% accuracy)
  • Step 2 LLM content analysis — reads document content (~95% accuracy)
  • Step 3 User confirmation — human-in-the-loop (100% accuracy)

Tested against production repos including Stripe, Cal.com, Supabase, and Vercel documentation.

Morning Briefing Personal

Automated daily email system that sends a personalized morning briefing at 6 AM PST every day, regardless of whether my computer is on. Claude generates the full HTML email using web search for live data, then sends via SendGrid. Content includes weather, industry news, market pulse, a "Great Mind of the Day" (alternating historical and living figures), and a patent/invention spotlight.

Each email is auto-summarized and appended to a running history file, giving the system persistent memory across executions despite being stateless per run. Runs on Railway as a cron job — costs under $1/month.
